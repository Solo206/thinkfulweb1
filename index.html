Is HTML considered a programming language?

No, HTML is a markup language. The "M" stands for "Markup". Generally a programming language allows you to describe some sort of process of doing something, whereas HTMl is a way of adding context and structure to text.

One of the commenters in Stack Overflow explained that HTML5+CSS3 could be considered Turing complete. Turing complete is according to wikipedia: a system of data-manipulation rules that can be used to simulate any single-taped Turing machine. In other words its a system that will find an answer or solve computational problems.

The prevailing thought however still remains that HTML by itself is not a programming language. A blog from Syracuse University succintly posits that HTML "is not an example of an executable script". It points out that HTML is used for structural purposes on a web page, not functional ones". Programming languages must have functional purposes, such as evaluative, computational or logical nature.

What is a tag?

According to w3chools.com, HTML tags can be identified with several significant features: surround keywords with angle brackets, come in pairs, consist of start/end tag or opening-closing pairs, and end tag is written with a slash. Wikipedia points out that tags are used to interpret content of pages. HTML describes structure of website semantically. Tags help parse content into an organized form to be digested by browser. Tags also can give meaning to specific portions of web page content and provide browsers/search engine extra information regarding how the content should be interpreted.

How can you tell the difference between an open and closing tag?

As stated before an open tag is a tag without a slash and a closing tag is one with a slash. It is signifies this is a parseable element from the rest of the html document. An example of an open tag is <p> and its closing tag is </p>. Everything inside that tag is content associated with that tag element.

Does the <!doctype> tag come before or after <html>?

<!DOCTYPE html> comes before html tags. It informs the browser what type of document its reading. This abbreviated document typing is a new feature of HTML5. Previous HTML versions had required whether the document was strict, transitional, frameset. In HTML5, the document type declaration only requires <!DOCTYPE html> to coded at the beginning of document.

Does information between <head> appear on rendered page?

No, information between head tags do not appear on the rendered page. According to w3school <head> element contains all head elements which include title for document, scripts, styles, meta information. Typically <head> is where meta information is located regarding how the document should be read, linked, and styled.

What does the <title> do?

The title tag typically represents text that will appear on the tab of your document page. It also will be the title of your bookmark for that particular page. According to w3schools, it defines the title of the browser toolbar, provides a title for the page when added to favorites and displays a title for the page in search-engine results

What goes inside the <body> tags of a page?

The main content of your document resides in the body of the page including articles, images, headers, footers, etc. W3schools defines <body> tag as the documents body that contains the contents of HTML document such as text, hyperlinks, images, tables, lists, etc.

What's a div?

<div> is a tag without semantic meaning but helps parse out content throughout the document. W3schools defines div tags as a division or section in an HTML document. It is a primary block-elemtn group that is used to format a document in conjunciton with CSS. From sitepoint.com/web foundations, div is a generic block-element that doesn't convey semantic meaning.

What's the difference between a block element(for instance, <div>) and an inline element (for instance <span>

Block element is element that takes up the entire width of the screen and does not allow for other elements to reside next to it. Inline elements allow other elements to be adjacent to it on a horizontal line. HTML Dogs website states that a span is an inline element that resides in a paragraph, whereas div is a block-line which takes up the entire line with line breaks before and after it. Inline elements allow text and other elements to wrap around it. It can be useful for specifying the alignment of that element according to surrounding elements.
