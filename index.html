How to style with CSS

There are 3 ways to style:

  • external
  • internal
  • inline
external styling
External styling is ideal when the style is applied to many pages. With an external style sheet, one file controls the entire look of the website. <link rel="stylesheet" href="stylesheet.css">
internal styling
Internal styling is used when a single document has a unique style. Style is defined in the head section of an HTML page <style> div{color:white;}</style>
inline stying
Inline styling loses many of the advantages of external style sheet, by mixing content with presentation. To use inline styles, add the style attribute to the relavant tag. Style attribute can contain any CSS property. <p style="color:white;margin-left:20px">This is a style.</p>

Styling occurs in the following precendence order from lowest to highest:

  1. Browser default
  2. External styling
  3. Internal styling
  4. Inline styling

Block-level elements

  • If no width is set, it will expand naturally to fill its parent container
  • Can have margins and or padding
  • If no height is set will expand naturally to fit its child elements
  • By default will be placed below previous elements in the markup assuming no floats or positioning on surrounding elements

Inline elements

  • Flows along with text content
  • Will not clear previous content to drop to the next line like block elements
  • Is subject to white space settings in CSS

Block vs Inline

Block elements begin on a new line stacking one on top of the other and occupy any available width. Block-level elements may be nested inside one another and may wrap inline-level elements. We'll most commonly see block level elements used for larger pieces of content such as paragraphs.

Iinline do not begin on a new line. They fall into normal flow of the document, lining up one after the other, and only maintain the width of their of their content. Inline level elements may be nested inside one another; however they cannot wrap block-level elements. We'll usually see inline level elements with smaller pieces of content.

Headings

Headings are block-level elements. Headings are also used to establish hierarchy, they are key identifiers used by users. Search engines also use them to index and determine content of page. Headings hsould be used in order that relavant to content of page. Subsequent headings within h1 sphere should use h2, h3, h4, h5, and h6. Headings have semantic value and should not be used to increase size of fonts.

Paragraphs

Headings are also followed by supporting paragraphs. Paragraphs are defined using the p block element.

Bold Text with Strong

Bold text places strong importance to it. Strong inline level element is used to give strong importance to text. <b> is used to stylistically offset text, for things that are not as important, but more for keywords. Gauge the usage based on the significance of the contextual meaning of words. If its actually important, defer to strong tag rather than b tag.

Target attribute

Target attribute is the ability to determine the location of where the link opens to when clicked. Links may open to same window or new windows or section of same window. To trigger the action of opening a link in a new windw, use the target="_blank". To open in same frame as clicked the link page, which is the default: target="_self". To open to the document of the parent frame, target="_parent". To open to the full body of window, target="_top". To open to named frame, target="framename"

Relative and absolute paths

Relative paths are typically restricted to their own website, such as for other pages or sections within the same web page. If the page in a different directory on the same site , then the relative path needs to corresponding to where it is in relation to the page of the link.

Absolute path requires the full domain name with the path directories. This is utiized in linking sites to other external sites.

Linking to parts of the same page

Hyperlinks can link to parts of same page. On-page links are first initiated by setting an id attribute with element that is link destination. Then reference the id with #idname in href field. Like so <a href="#top">. top

Building structure

Header element

Header element is used to identify the top of a page, article, section or other segment of a page. Header element may include heading, introductory text, and even navigation. Remember not to get Not to confuse header, head, and h1-h6 tags. Head elements are usually not dispalyed on page and used to outline metadata like title, and links to other files. h1-h6 elements are used to designate multiple level of text headings and hierarchy of structure within page.

Navigation element

Nav element identifies a section of major navigational links on page. Nav element should be reserved for primary navigation sections such as global navigation like menus, table of contents, previous/next links. Most commonly links include links to other parts of page or other pages in same website. Single links should be avoided in nav elements

Article element

Article elements are used to identify section of independent, self-contained content that may be independently distributed or reused. Often, they are used to mark up blog posts, newspaper articles , suer-submitted content. The article element should be independent of rest of page and should be readily understood by itself, without confusion.

Section element

Section elements are used to identify thematic groupings of content, which generally include a heading. The grouping of content wihtin section element may be generic in nature, but useful to identify all the content related. Section element is commonly used to break up and provide hierarchy to a page.

Aside element

Aside element holds content such as sidebars, inserts, brief explanations that is tangentially related to content surrounding it. e.g. identify other works by author of website. Aside is a block level elements as such will appear on full line, occupying width of page or parent element.

Footer element

Footer element identifies closing or end of page, article, section or other segment of page. Generally footer element is found at bottom of its parent. Content within footer element should be relative information and should not diverge from the document or section it is included within.
